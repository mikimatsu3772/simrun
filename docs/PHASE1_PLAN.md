# Phase 1 — 育成コア 実装計画

## ゴール

「移動距離 → TP → トレーニング → ステータス成長 → 引退」の1世代サイクルが
端末内で完結して動くこと。レース・継承・収集は次フェーズ。

## 距離ソースの抽象化

ヘルスケアAPI(HealthKit / Health Connect)は実機専用のため、
距離取得を `DistanceSource` インターフェースで抽象化する:

```
DistanceSource(抽象)
├── HealthDistanceSource   … 実機用(health パッケージ)
└── MockDistanceSource     … 開発・テスト用(手動入力 / 自動生成)
```

開発中は macOS デスクトップ / Chrome で MockDistanceSource を使い、
UI とゲームロジックを先に完成させる。

## データモデル(Phase 1 スコープ)

### Runner
- id, name, 誕生日(世代開始日)
- タイプ(スピード型 / スタミナ型 / 根性型 / バランス型)
- 得意距離(短 / 中 / 長)
- ステータス: speed / stamina / power / guts / wisdom
- 成長傾向: 各ステータスの成長係数
- 状態: 現役 / 引退
- ※ 将来のゴースト対戦を見越して JSON シリアライズ可能にする

### ActivityRecord(移動記録)
- 日付、距離(m)、時間帯区分(朝 / 昼 / 夜)
- 変換済みTP、変換レート

### TrainingSession
- 対象ステータス、消費TP、成長量、実施日時

## コアロジック

1. **距離→TP変換**: 1km = 10TP を基準。時間帯ボーナス(朝+10%等)
2. **トレーニング**: TPを消費して任意のステータスを成長。成長量 = 基本値 × 成長傾向係数
3. **世代タイマー**: 世代開始から14日で引退イベント発生
4. **引退**: ステータスを凍結して殿堂入り(継承ロジック自体は Phase 3)

## 画面構成(Phase 1)

1. **ホーム**: 現在のランナー、今日の移動距離、所持TP、世代の残り日数
2. **トレーニング**: ステータス一覧とTP割り振り
3. **記録**: 日々の移動ログ
4. **デバッグパネル**(開発ビルドのみ): 距離の手動注入、日付送り

## 技術選定

- 状態管理: riverpod
- 永続化: drift(SQLite)
- 距離取得: health(実機のみ)
- テスト: ロジック層は純 Dart でユニットテスト可能に保つ
