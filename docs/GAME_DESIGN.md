# simrun — ゲームデザインドキュメント

## コンセプト

**現実の移動距離でランナーを育てる育成シミュレーションゲーム。**

プレイヤーが日常で歩いた・走った距離がトレーニングポイント(TP)に変換され、
ドット絵のランナーを育成し、レースに出場させ、引退後は次世代へ能力を継承していく。

- プラットフォーム: iOS / Android(Flutter)
- 距離計測: HealthKit / Google Fit(Health Connect)連動 — 歩数・距離APIを軸にする
- 通信構成: まずオフライン完結(端末内保存)。対人要素は後からサーバー追加できる設計にしておく
- ビジュアル: ドット絵・レトロ風
- レース: 自動シミュレーション観戦型(ウマ娘・ダビスタ系)

## コアループ

```
現実で移動する
  → 距離がTPに変換される
  → TPを使ってランナーをトレーニング(ステータス育成)
  → レースに出場(自動シミュ観戦)
  → 結果に応じて報酬・実績
  → 約2週間で引退 → 能力を次世代へ継承
  → 新ランナーで周回(より強く、より遠くへ)
```

## 育成システム

### 1世代 = 約2週間

日常5〜6km歩く人で約70〜80km/世代。飽きる前に世代交代の楽しみが来るテンポ。

### ステータス(案)

| ステータス | 役割 |
|---|---|
| スピード | 短距離・スパートの強さ |
| スタミナ | 長距離の持久力 |
| パワー | 坂・荒れた展開への対応 |
| 根性 | 終盤の粘り、接戦時の勝負強さ |
| 賢さ | 作戦の実行精度、位置取り |

### 個性(素質)

ランナーごとに生成される個性:

- **タイプ**: スピード型 / スタミナ型 / 根性型 / バランス型
- **得意距離**: 短距離(〜5km) / 中距離(〜ハーフ) / 長距離(フル)
- **成長傾向**: 各ステータスの成長しやすさ(継承で変化)

### 時間帯・状況ボーナス

走った(歩いた)現実の状況で成長にフレーバーがつく:

- 朝の移動 → 「集中力」ボーナス(賢さ寄り)
- 雨の日 → 「根性」ボーナス
- 長時間連続の移動 → スタミナ寄りに成長

## 世代交代・継承システム(周回の柱)

- 約2週間で引退イベント。引退ランナーは**コーチ**となり殿堂入り
- 継承要素:
  - ステータスの一部を素質として次世代に遺伝
  - コーチスキル: 殿堂入りランナーが次世代のトレーニング効率を上げる
  - 二つ名・実績: 「雨の王者」等の称号が血統に残る
- 世代を重ねるほど初期素質の上限が上がる → 長期の成長実感

## レース・大会(中期目標の柱)

- 累積距離で出場権を獲得: 5km大会 / 10km / ハーフ / フルマラソン相当
- 自動シミュレーション観戦: ドット絵ランナーたちがレース展開を繰り広げる
  - 事前に作戦指示(逃げ/先行/差し/追込 等)
  - 展開・天候・コース適性で結果が揺れる
- 大会グレード制: ローカル大会 → 地方大会 → 全国 → 世界
- 世代内で勝ち進むことが引退時の継承ボーナスに直結

## 対人要素(ゴースト対戦)

オフライン期はこう成立させる:

- **過去の自分のゴースト**: 歴代の殿堂入りランナーとレースで再戦
- **NPCゴースト**: プリセットのライバルキャラ(ストーリー性を持たせる)

将来サーバー追加時:

- 他プレイヤーのランナーデータ(ゴースト)をダウンロードして非同期対戦
- シーズン制ランキング

設計上の注意: ランナーのデータモデルは最初から「シリアライズして他端末で再生できる」形にしておく。

## 収集・装備強化(コツコツの柱)

- 移動距離に応じてアイテム獲得(ランダムドロップ):
  - シューズ、ウェア、サプリ、トレーニング器具
- 装備でステータス補正・トレーニング効率UP
- レアリティとセット効果
- 実績バッジ: 累積100km、早朝ラン、雨の日ラン、連続7日移動 など

## リテンション設計まとめ

| 時間軸 | 楽しみ |
|---|---|
| 毎日 | 移動→TP獲得→トレーニング、アイテムドロップ |
| 毎週 | 週間ミッション、大会出場 |
| 2週間 | 世代交代・継承の節目 |
| 長期 | 血統の成長、殿堂・実績収集、大会グレード制覇 |

## 技術構成

- **Flutter** (iOS / Android)
- 距離取得: `health` パッケージ(HealthKit / Health Connect)
- ローカル保存: `drift`(SQLite)または `hive`
- 状態管理: `riverpod`
- ドット絵描画: Flutter標準ウィジェット + `flame`(レースシーン)を検討

## 開発フェーズ(案)

1. **Phase 1 — 育成コア**: 距離取得 → TP変換 → ステータス育成 → 1世代サイクル
2. **Phase 2 — レース**: 自動シミュレーション + 観戦演出(ドット絵)
3. **Phase 3 — 継承**: 世代交代、殿堂、コーチシステム
4. **Phase 4 — 収集**: 装備・アイテム・実績
5. **Phase 5 — 対人**: NPCゴースト → (将来)サーバー対戦
